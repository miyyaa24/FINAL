파이널 프로젝트 명명규칙

■공통 규칙
1. 숫자로 시작해서는 안된다.
2.특수문자는 '_'만 허용한다
3. 한국어로 발음나는데로 네이밍을 하지 않는다.

■클래스 명명규칙

파스칼표기법을 사용한다.
ex) HelloWorld

■메소드 명명규칙

카멜표기법을 사용한다.
ex) findData()

■ 변수 명명규칙
1. 변수와 파라메터에는 카멜표기법을 사용한다.
2. 변수에 약어 사용을 지양하고 의미를 충분히 담는다
3. 한글자로 된 이름을 사용하지 않는다.
4. boolean타입 변수는 네이밍을 is로 시작한다. ex) isFind

■VO 명명규칙

1.카멜표기법을 사용한다
2.문자간 '_'은 최대한 지양한다.
3. VO변수 선언시 주석으로 어떤변수인지 설명 ex) Private String name;   // 이름

■ DB 명명규칙
1. 모든 네이밍을 대문자로 사용한다.
2. 문자와 문자 사이에는 '_'를 사용한다.
3.가능하면 단어를 줄여쓰지 않는다.
ex) MID_NM (X) --> MIDDLE_NAME (O)
